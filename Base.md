Без проблем, давай сделаем этот «пропускной пункт». Ниже представлен простой Bash-скрипт, который выполняет твою задачу.

Я добавил скрытие ввода пароля (флаг `-s`), чтобы всё выглядело по-взрослому и никто не подсмотрел символы через плечо.

---

### Скрипт проверки доступа

```bash
#!/bin/bash

# Укажите здесь правильные данные для входа
VALID_LOGIN="admin"
VALID_PASSWORD="password123"

# Проверяем, передан ли логин как аргумент
if [ -z "$1" ]; then
    echo "Ошибка: Введите логин как параметр. Пример: $0 имя_пользователя"
    exit 1
fi

# Проверка логина
if [ "$1" == "$VALID_LOGIN" ]; then
    # Запрашиваем пароль (флаг -s скрывает ввод в терминале)
    read -sp "Введите пароль: " user_pass
    echo # Перенос строки после ввода пароля

    # Проверка пароля
    if [ "$user_pass" == "$VALID_PASSWORD" ]; then
        echo "Доступ разрешен. Ваш секретный код: gl0kayakyzdr@"
    else
        echo "неправильный пароль - ыускуе"
    fi
else
    echo "Ошибка: Пользователь с таким логином не найден."
fi

```

---

### Как это запустить:

1. **Создай файл:** Скопируй код и сохрани его под именем `check_login.sh`.
2. **Дай права на выполнение:**
```bash
chmod +x check_login.sh

```


3. **Запусти скрипт, передав логин:**
```bash
./check_login.sh admin

```



### Небольшие пояснения:

* **`$1`**: Это первый аргумент, который ты пишешь в консоли после имени скрипта.
* **`read -sp`**: Позволяет пользователю вводить текст, не отображая его на экране. Полезно для безопасности.
* **`ыускуе`**: Это слово «secret», набранное в русской раскладке. Забавный штрих!
